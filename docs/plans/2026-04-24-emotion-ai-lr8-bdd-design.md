# ЛР8: Emotion AI — дизайн BDD-решения

## Контекст
Базой служит проект ЛР7: Node.js + Express + веб-интерфейс + Playwright UI-тесты.

## Цель ЛР8
Добавить приемочное тестирование в стиле BDD:
- 3-5 Gherkin-сценариев;
- позитивные и негативные кейсы;
- минимум 1 Scenario Outline с Examples;
- автоматизация шагов.

## Выбранный подход
Cucumber.js + Playwright поверх текущего UI.

## Компоненты
- `features/*.feature` — приемочные сценарии
- `features/step-definitions/*.js` — автоматизация шагов
- `features/support/world.js` — контекст сценариев
- `features/support/hooks.js` — lifecycle (browser/context/server)
- `cucumber.js` — конфигурация раннера
- `tests/fixtures/*` — тестовые изображения для сценариев эмоций

## Набор сценариев
1. Успешная авторизация.
2. Ошибка авторизации при неверном пароле.
3. Регистрация нового пользователя и вход.
4. Смена роли пользователя администратором.
5. Scenario Outline: распознавание эмоции по разным именам файлов.

## Поток выполнения
1. Cucumber читает `.feature`.
2. Шаги маппятся на функции в `step-definitions`.
3. Playwright выполняет действия в браузере.
4. Проверки в шагах валидируют ожидаемое поведение.

## Ограничения
- Для первого запуска нужен доступ к npm registry (установка `@cucumber/cucumber`).
- Состояние пользователей in-memory, меняется во время тестов.

## Критерии готовности
- Все сценарии из `features/` выполняются через `npm run test:bdd`.
- Есть отдельная инструкция по запуску и отчет ЛР8.
