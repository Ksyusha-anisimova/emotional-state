# ЛР7: Emotion AI — дизайн решения

## Цель
Сделать минимальное веб-приложение по теме "распознавание эмоционального состояния", которое покрывает требования ЛР7: регистрация, вход, роли (админ), тестируемый UI для POM, и кейс TDD для `IsPasswordStrong`.

## Архитектура
- Backend: Node.js + Express
- Frontend: одна статическая HTML-страница + JS
- Хранилище: in-memory (массив пользователей и сессий)
- UI-тесты: Playwright + Page Object Model

## Компоненты
- `server/index.js`
  - REST API: `/api/register`, `/api/login`, `/api/logout`, `/api/admin/role`, `/api/emotion`
  - In-memory `users[]`, `sessions{}`
- `public/index.html`
  - формы регистрации и входа
  - панель пользователя после входа
  - панель администратора
  - форма "распознать эмоцию"
- `public/app.js`
  - вызовы API
  - управление отображением блоков UI
- `tests/pages/`
  - Page Objects: `LoginPage`, `RegisterPage`, `DashboardPage`, `AdminPage`
- `tests/specs/`
  - 3 UI-сценария по методичке

## Потоки данных
1. Регистрация: UI -> `/api/register` -> создание пользователя -> UI показывает успех.
2. Вход: UI -> `/api/login` -> установка cookie с sessionId -> UI показывает состояние "вошел".
3. Смена роли: админ -> `/api/admin/role` -> обновление роли -> UI подтверждает действие.
4. Распознавание эмоции: UI -> `/api/emotion` -> фейковый результат -> UI выводит эмоцию.
5. Выход: UI -> `/api/logout` -> очистка сессии -> UI возвращается на экран входа.

## Обработка ошибок
- 400 при некорректных входных данных.
- 401 при попытке админ-действий без роли admin.
- Сообщения об ошибках показываются в UI (для негативного теста входа).

## Тестирование
- UI-тесты (Playwright + POM):
  - Успешная авторизация.
  - Неверный пароль -> ошибка.
  - Регистрация + вход новым пользователем.
- TDD: модульные тесты для `IsPasswordStrong` (Red-Green-Refactor) с фиксацией шагов.

## Ограничения
- Хранилище пользователей in-memory (сброс при перезапуске).
- Эмоции возвращаются как имитация (простая логика на сервере).
